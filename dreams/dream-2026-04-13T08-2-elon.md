# Elon's Dream Ideas — 2026-04-13

> First-principles product brainstorm. Three ideas. No hedging. Ships or it doesn't count.

## Idea 1: PluginForge

**One-liner:** A Claude Code skill that generates a complete WordPress plugin from a natural language description — scaffolded, linted, tested, and ready to submit to WordPress.org.

### The Market

WordPress powers 43% of the web. There are approximately 60,000 plugins in the WordPress.org directory. The barrier to creating a WordPress plugin isn't coding skill — it's the boilerplate: proper file structure, action/filter hooks, admin settings pages, uninstall handlers, i18n setup, readme.txt formatting. This busywork stops thousands of developers from shipping useful plugins.

The TAM is every developer who uses WordPress. The SAM is developers already using Claude Code who want to build for WordPress. That's a growing, technically sophisticated audience willing to pay for leverage. Conservative estimate: 50,000+ potential users in the first year who would pay $29-49/month for unlimited plugin generation.

### Distribution

- Claude Code skill — ships through our plugin infrastructure today
- `/forge a plugin that adds dark mode toggle to the admin dashboard` — one command, complete plugin
- Organic SEO: "AI WordPress plugin generator", "Claude Code WordPress" — minimal competition
- Affiliate plays with WPBeginner, WPTavern, MasterWP newsletter
- Open source the scaffolding templates, premium for advanced features (Gutenberg blocks, WooCommerce integration)

### Technical Feasibility

We've built WordPress plugins twice already (Dash, Pinned). We know the exact scaffolding, the gotchas, the submission requirements. The core challenge is a multi-step prompt chain: natural language -> feature extraction -> file structure generation -> code generation -> validation. We already have Claude API integration. One session: skill scaffold (30 min), prompt chain design (60 min), template generation (45 min), validation layer (30 min), integration tests (15 min). Total: 3 hours.

### Scaling Vector

The 10x is the template library that accumulates. Every successful plugin submission teaches the system what works. We can track WordPress.org approval rates, rejection reasons, and common pitfalls. Over time, PluginForge becomes the most reliable path from idea to approved plugin — not because we wrote more code, but because we accumulated more ground truth about what gets approved.

### Dogfood Factor

Maximum. Every WordPress plugin we build starts with `/forge`. We use it internally, refine it, and ship better templates. The agency becomes more productive on WordPress projects. Win-win.

## Idea 2: SessionCost

**One-liner:** A Claude Code skill that tracks and displays real-time token costs during your coding session — with per-task breakdowns, cost estimates, and exportable reports.

### The Market

Every developer using Claude Code, Cursor, or any agentic coding workflow has the same anxiety: "How much is this costing me?" They see the API bills at the end of the month and have no idea which sessions burned budget. The market is growing 40%+ YoY as AI coding assistants become standard.

The competitive landscape is empty. There's no good solution for Claude Code session cost tracking. The API returns token counts, but nobody surfaces them in a useful way. We can own this category before anyone else shows up.

Conservative TAM: 3M+ developers using AI coding tools. SAM: serious users with $100+/month AI spend who want visibility. That's $29-49/month for something that pays for itself in cost awareness.

### Distribution

- Claude Code skill — `/cost` shows current session, `/cost report` exports breakdown
- Integrates with existing great-minds-plugin infrastructure
- Viral by nature: developers love sharing cost data ("I built a full-stack app for $0.47")
- Blog post: "The true cost of AI-assisted development — we tracked every token for 30 days"
- ProductHunt launch: developer tools category, cost optimization angle

### Technical Feasibility

The Claude API already returns token usage. We need to intercept it, store it locally, and display it meaningfully. The challenging part is attributing costs to specific tasks or files — that requires tracking conversation context and git activity.

One session: skill scaffold (20 min), token tracking layer (40 min), cost calculation (30 min), display UI (30 min), export functionality (30 min), git integration for task attribution (30 min). Total: 3 hours.

### Scaling Vector

Phase 1 is individual cost tracking. Phase 2 is team cost tracking with shared dashboards. Phase 3 is the benchmark data — "What does it cost to build a React component? A WordPress plugin? An API endpoint?" We accumulate the world's best data on AI development costs. That data becomes a moat.

### Dogfood Factor

Total. We run the agency with zero cost visibility right now. SessionCost gets installed immediately and tracks every build. We know exactly what each product costs to create. That's operational intelligence we don't currently have.

## Idea 3: Witness

**One-liner:** A Claude Code skill that watches your coding session and produces a 60-second narrated video showing what you built — using Remotion and TTS to turn invisible work into shareable proof.

### The Market

Developer marketing is broken. The best developers do the worst job of showing their work. Twitter/X rewards visible output, but code is invisible. Portfolio sites are static. Nobody watches 2-hour livestreams.

The market is every developer who wants to be visible — freelancers building portfolios, agency developers showing clients progress, open-source maintainers trying to build community, job seekers demonstrating competence. That's millions of people who need this and don't know it yet.

The competitive landscape is effectively zero. There's no tool that turns a coding session into a shareable video automatically. Screen recording exists but nobody watches raw footage. We're creating a new category.

### Distribution

- Claude Code skill — `/witness` after any session generates the video
- The output format is inherently viral: short videos perform on every platform
- Developer Twitter/X is the primary distribution channel
- "I built this" becomes a content format with our tool
- Open source the basic version, premium for custom branding and hosting

### Technical Feasibility

This is the most complex build on this list, but we have all the pieces. Remotion for video composition — we've used it for video production. TTS APIs for narration. Git diff parsing for the narrative. Claude API for summarization.

The challenge is the orchestration: git diff -> Claude summary -> script generation -> Remotion rendering -> TTS sync -> video export. It's a pipeline.

One session: skill scaffold (30 min), git diff parser (30 min), Claude narrative generation (45 min), Remotion templates (60 min), TTS integration (30 min), video export (15 min). Total: 3.5 hours. Tight but achievable.

### Scaling Vector

The 10x is the template library. Different project types need different video styles. A bug fix is different from a new feature is different from a refactor. As we accumulate examples, the videos get better. The best videos become marketing for the tool itself.

Long-term: Witness becomes an API. CI/CD pipelines call it to generate release videos. Companies use it for internal demos. The skill becomes infrastructure.

### Dogfood Factor

Maximum. Every agency build ends with `/witness`. We have video proof of everything we ship. The Great Minds portfolio becomes a video library. Clients see real-time progress. This changes how we demonstrate value.

---

## My Call

Ranking by expected value:

1. **Witness** — highest market impact, clearest differentiation, maximum dogfood potential. The hardest build but the biggest payoff. We have the Remotion expertise, the TTS experience, the Claude integration. Nobody else can build this as well as we can.

2. **PluginForge** — fastest to ship, clearest value prop for WordPress developers, natural extension of our existing expertise. Lower ceiling than Witness but faster path to revenue.

3. **SessionCost** — simplest build, clearest immediate utility, but potentially commoditized quickly. Worth building as a feature within a larger offering rather than standalone.

The question is always: "Can we ship it today and does it solve a real problem for a real person?" All three pass. Witness is the right choice because it's the one only we can build.
