## Executive Summary

## Product-by-Product First-5-Minutes Audit

### 1. LocalGenius (localgenius.company)

**First Impression (0-30 seconds):**
- ✅ Strong tagline: "You didn't open a restaurant to do marketing. So stop doing it."
- ✅ Clear value proposition immediately visible
- ✅ "No credit card required" reduces friction

**Confusion Points (30 seconds - 5 minutes):**
- ⚠️ **What happens after I sign up?** The conversational AI interface is mentioned but not shown. Users don't know what they're getting into.
- ⚠️ **"5-minute website" feels too good to be true.** No before/after examples shown.
- ⚠️ **Who is this actually for?** Restaurants are mentioned, but the product claims to serve all local businesses. Mixed messaging.

**Emotional Journey:**
- Intrigue → Skepticism → Uncertainty → "Maybe I'll try it later"

**The 5-Minute Fix:**
1. **Show the conversational UI** in action (GIF or video). Users need to see what "conversational marketing" looks like before they'll trust it.
2. **Feature 3 real businesses** with names and photos: "Maria's Café generated 47 Google reviews in 60 days."
3. **Pick one vertical** for the landing page. A restaurant owner doesn't want to feel like they're using a generic tool.

### 2. Dash (WordPress Command Palette)

**First Impression (0-30 seconds):**
- ✅ Clear functionality: "Press Cmd+K to search and navigate"
- ✅ Immediate value for power users

**Confusion Points (30 seconds - 5 minutes):**
- ⚠️ **"Why do I need this?"** WordPress already has a command palette. The differentiation isn't clear immediately.
- ⚠️ **No installation preview.** Users don't know if this will conflict with existing plugins or how it integrates with their admin.
- ⚠️ **Who uses this?** No testimonials or usage numbers to create trust.

**Emotional Journey:**
- Recognition → "Do I need this?" → "I'll bookmark it and maybe install later"

**The 5-Minute Fix:**
1. **Lead with the pain.** "WordPress's built-in command palette misses 80% of your admin pages. Dash indexes everything."
2. **Side-by-side comparison** video: WordPress default vs. Dash (10 seconds max).
3. **Show recent items and commands.** These are killer features buried in the spec sheet.
4. **"Used by X developers"** social proof, even if X is small.

---

### 3. Pinned (WordPress Sticky Notes)

**First Impression (0-30 seconds):**
- ✅ Concept is instantly understandable: sticky notes on your dashboard

**Confusion Points (30 seconds - 5 minutes):**
- ⚠️ **"Is this for me or my team?"** The solo vs. team use case isn't clear.
- ⚠️ **@mentions and acknowledgments sound powerful** but users won't discover them until they're already using the product.
- ⚠️ **What happens when I have 50 notes?** Users fear clutter. The auto-expiration feature should be front and center.

**Emotional Journey:**
- "Oh cute" → "But do I need this?" → "My dashboard is already cluttered" → Bounce

**The 5-Minute Fix:**
1. **Lead with the team use case.** "Stop losing client feedback in Slack." Solo users can still use it, but teams have the pain.
2. **Show auto-expiration upfront.** "Notes fade away after 7 days. No note graveyards."
3. **Demo the @mention notification.** This is the stickiest feature (pun intended) and it's hidden.
4. **Onboarding tooltip inside the dashboard** that walks users through creating their first note.

### 4. Great Minds Plugin

**First Impression (0-30 seconds):**
- ✅ Ambitious and exciting: "14 AI agents build your product"
- ⚠️ **Intimidating.** For a new user, "just drop a PRD" assumes they know what a PRD is and how to write one.

**Confusion Points (30 seconds - 5 minutes):**
- ⚠️ **"How do I actually start?"** The npx installation is clear, but what happens next is opaque.
- ⚠️ **17 skills listed** but no guidance on which one to use first.
- ⚠️ **Agent names (Steve, Elon, Jensen)** are fun but confusing. What does Steve actually do versus Elon?
- ⚠️ **"155 memories indexed"** — Cool for developers, meaningless for first-time users.

**Emotional Journey:**
- "Wow this is ambitious" → "Wait, how do I use this?" → "I need to read documentation" → "Maybe later"

**The 5-Minute Fix:**
1. **One-minute quickstart video.** Show a PRD going in, show deliverables coming out.
2. **PRD template included.** Don't assume users know how to write one. "Here's a 10-line PRD for a simple landing page."
3. **First command should be obvious.** `/agency-launch` should be the only command a new user needs to know.
4. **Agent explainer (30 seconds).** "Steve handles design. Elon handles engineering. Jensen reviews strategy. That's all you need to know."

---

### 5. Shipyard AI (www.shipyard.company)

**First Impression (0-30 seconds):**
- ✅ Clear value: PRD to production
- ✅ Transparent pricing (token-based) is refreshing
- ✅ "No meetings" appeals to a specific persona

**Confusion Points (30 seconds - 5 minutes):**
- ⚠️ **"What's a PRD?"** Again, we assume users know. Many don't.
- ⚠️ **Pricing tiers show tokens, not dollars.** Confusion: "How much will my project actually cost?"
- ⚠️ **No portfolio of shipped sites.** "100% ship rate" is a claim without proof.

**Emotional Journey:**
- Intrigue → "This sounds efficient" → "But how much will it cost me?" → "Where's the portfolio?"

**The 5-Minute Fix:**
1. **Token-to-dollar converter.** "A 10-page site typically costs $X" or an instant calculator.
2. **Portfolio carousel.** Show 3-5 shipped sites with links.
3. **"What's a PRD?" tooltip.** Don't lose users who don't speak product manager.
4. **Example PRD submission.** "Here's what a client sent us, and here's what we shipped."

## Cross-Portfolio Pattern

### The Universal Problem: **Assumed Knowledge**

Every product assumes users already understand:
- What the product category is
- Why they need it
- How to get started

**This is losing you users.** People bounce not because the product is bad, but because they don't understand it fast enough.

### The Fix: **Show, Don't Tell**

| Product | Current State | Recommended State |
|---------|--------------|-------------------|
| LocalGenius | Describes conversational UI | Shows conversational UI in action |
| Dash | Lists features | Compares before/after search |
| Pinned | Lists capabilities | Demos team acknowledgment flow |
| Great Minds | Lists 17 skills | Shows one PRD → deliverable journey |
| Shipyard | Lists process stages | Shows example project with cost |

## Top 3 First-5-Minutes Improvements

| Priority | Product | Change | Impact |
|----------|---------|--------|--------|
| 1 | Great Minds | 60-second quickstart video + PRD template | Removes biggest barrier to adoption |
| 2 | LocalGenius | Interactive demo or GIF of conversational UI | Converts curiosity to trials |
| 3 | Shipyard | Token calculator + 3 portfolio examples | Builds trust and answers "how much?" |

## Emotional Truth

**People don't buy products. They buy confidence that the product will work for them.**

Right now, your products create intrigue but not confidence. The fix is simple: show the transformation. Before → After. Input → Output. Pain → Relief.

Every landing page should answer three questions in 30 seconds:
1. **What is this?** (One sentence)
2. **What does it look like?** (Visual proof)
3. **How do I start?** (One clear action)
