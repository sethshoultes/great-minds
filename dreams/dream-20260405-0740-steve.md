## Idea 1: Pulse

### One-liner
A WordPress plugin that turns your site's analytics into a single living number — the heartbeat of your business — and whispers to you in plain English when something is wrong.

### The User
WordPress site owners who run real businesses — WooCommerce stores, membership sites (MemberPress), course platforms. They are not data scientists. They open Google Analytics, feel nothing, close the tab. They are flying blind and they know it but they have no idea what to do about that. Pulse is for them.

Right now they need five separate tools to answer: "Is my business healthy today?" Pulse answers that in one glance, without a single dashboard.

### Distribution
- WordPress.org plugin directory (free tier, premium AI tier)
- MemberPress ecosystem — we have direct access and deep familiarity
- WooCommerce ecosystem — enormous install base, almost no beautiful analytics tools
- Launch post on the Great Minds blog with a Remotion demo video
- "Site heartbeat" is a single shareable image — viral by design, like sharing your Spotify Wrapped

### Why It's Insanely Great
Every other analytics tool shows you what happened. Pulse tells you what it *means*.

The number is 0 to 100. Green, yellow, red. One look and you know. Below the number: three sentences. "Revenue is up 12% this week. Churn spiked yesterday — 3 cancellations, all from the same traffic source. Your best-performing page got no traffic today." That's it. No charts. No tables. No export to CSV.

The insight that used to require a data analyst costs $9/month and arrives in your WordPress admin like a gentle tap on the shoulder. People will feel cared for. That is rare.

### Build Plan
**Steve:** Designs the single-number UI, the color language, the three-sentence insight format. Writes the copy. Defines what "healthy" means as a weighted score formula.

**Elon:** Engineers the data aggregation layer — WooCommerce + MemberPress hooks, the scoring algorithm, the Claude API call for plain-English summarization. Defines the REST endpoint.

**Sub-agents:**
- Designer: Admin widget mockup, the "Pulse number" visual component
- Engineer: PHP plugin scaffold, data collectors, AI summarization call
- Copywriter: Onboarding copy, empty states, the tone of every AI-generated insight

**Session flow:** PRD -> debate score formula and insight format -> build plugin scaffold -> wire data collectors -> Claude summarization -> admin widget UI -> QA -> ship to GitHub

### Dogfood Factor
We wire this into every one of our six repos immediately. Phil runs the heartbeat — Pulse is the human-readable version of that. We use it on LocalGenius, on the Great Minds site, on every client site we ever touch. This is infrastructure for us, not just a product.

## Idea 2: Narrator

### One-liner
A Claude Code plugin skill that watches your git commits and automatically writes a living changelog in your voice — not technical release notes, but a story of what you built and why.

### The User
Indie developers and small agencies who ship constantly but never document their journey. They have beautiful products and no way to share the story behind the decisions. Their changelog is a git log. Their "about" page is a placeholder. They built something remarkable and nobody knows.

The user is us. Every developer who ships alone or in a small team and loses the narrative thread of their own work.

### Distribution
- Claude Code plugin (skill) — published to the Great Minds Plugin, distributed through the 12-skill ecosystem we already have
- GitHub-native: works on any repo, any language
- Blog post: "We let Claude write our changelog for six months — here's what happened"
- Developer Twitter/X — "your commits are a diary, Narrator reads it to you" is a shareable hook

### Why It's Insanely Great
Every great product has a story. The iPhone wasn't just specs — it was a narrative. But developers ship constantly and never tell the story of what they were thinking when they made a decision. Six months later they can't even remember why they rewrote that component.

Narrator reads your commits and writes two things: a human changelog entry ("This week we quietly fixed the thing that was making mobile users angry — a 12-line change that took three days to find") and a brief note on the decision behind it ("We chose to rebuild from scratch rather than patch because the patched version would have collapsed under load"). It's a diary you never had to write.

The output is a `STORY.md` that grows with the project. Beautiful. Shareable. The kind of thing you'd put on your product's homepage. It makes developers feel like their work *mattered* — because the story of the work is as important as the work itself.

### Build Plan
**Steve:** Defines the narrative format — the tone, the structure of a changelog entry, what gets included versus ignored. Writes the prompts that produce beautiful prose instead of dry summaries. Decides what "in your voice" means and how to calibrate it.

**Elon:** Engineers the git integration — commit parsing, diff summarization, frequency logic (weekly rollup vs individual commits). Defines the Claude API prompt chain. Handles the STORY.md write-back.

**Sub-agents:**
- Claude API Engineer: Prompt design for commit-to-narrative, voice calibration from existing STORY.md
- Plugin Engineer: Claude Code skill scaffold, git hooks, `narrator` slash command
- Copywriter: The example outputs that make developers say "I want that"

**Session flow:** Define output format -> build git parser -> design prompt chain -> Claude API integration -> skill scaffold -> slash command -> test on great-minds repo -> ship

### Dogfood Factor
We run it on all six repos from day one. The Great Minds changelog *becomes* Narrator's own story. We use the output as blog post drafts. The agency's memory system gets richer automatically — every commit tells us something we can learn from.

## Idea 3: Boardroom

### One-liner
A web app that lets you drop any decision — a job offer, a business pivot, a product launch — into a room with five great minds who argue it from every angle, then hand you a one-page verdict.

### The User
Founders, product leads, and anyone who needs to think harder about a consequential decision but doesn't have a board of advisors. They read about the Great Minds Agency and want that — but for their own decisions, not their code.

This is the democratization of what we do internally every day. We debate with Steve and Elon before we build anything. Most people make big decisions alone, with their own biases, in a vacuum.

### Distribution
- Standalone Next.js web app deployed on Vercel (we have the infrastructure)
- greatminds.company as the launch platform — we already have the brand
- Productized version of a concept people already understand: "get feedback from different perspectives"
- Launch with five pre-built panels: Startup Founder, Warren Buffett, Contrarian Analyst, User Advocate, Devil's Advocate
- Shareable verdict page — the output is designed to be screenshot and posted

### Why It's Insanely Great
Most advice tools give you one perspective. One AI chatbot. One point of view. The entire insight of the Great Minds Agency is that *conflict between smart perspectives* produces better thinking than any single genius.

Boardroom puts five distinct minds in a room — not sycophants, not generic advice, but characters with genuine points of view that contradict each other. Warren's lens is always unit economics. The User Advocate always asks "but what does the customer actually feel?" The Contrarian says "here's why you're wrong." The synthesis at the end is a one-page verdict written in the style of a supreme court opinion: "The majority recommends X. The dissent argues Y. If condition Z, reconsider."

People will feel less alone in their hard decisions. That is something no tool currently delivers.

### Build Plan
**Steve:** Designs the panel composition, the UX of submitting a decision, the format of the verdict page. Makes it feel like walking into an actual boardroom, not a chatbot. Defines the five default personas and their argumentative styles.

**Elon:** Engineers the multi-agent debate architecture — parallel Claude calls for each persona, synthesis layer, the streaming UI that shows the debate unfolding in real time. Defines the data model for decisions and verdicts.

**Sub-agents:**
- Next.js Engineer: App scaffold, decision submission form, streaming debate UI, shareable verdict page
- Prompt Engineer: Five persona system prompts, synthesis prompt, the "verdict" format
- Designer: The boardroom visual metaphor — table, seats, verdict document aesthetic
- Copywriter: Onboarding, empty state, example decisions that make the value obvious immediately

**Session flow:** PRD debate -> lock five personas and verdict format -> Next.js scaffold -> multi-agent API layer -> streaming UI -> verdict page -> shareable link -> deploy to Vercel -> ship

### Dogfood Factor
Every Great Minds Agency decision goes through Boardroom before we build. PRD evaluation, tech stack choices, pricing decisions — all of it. We are the first users and the most demanding ones. The product gets better every time we use it.

## My Verdict

These three products are not the same kind of thing. **Pulse** is infrastructure — it solves a problem every WordPress business owner has today, right now, and it's distributable through ecosystems we already live in. **Narrator** is a gift to developers — it gives their work a voice and costs almost nothing to distribute through our existing plugin. **Boardroom** is the most ambitious: it's the agency's core insight packaged for the world.

If I had to pick one to build tomorrow, it's **Boardroom**. It's the purest expression of what we do. It makes the Great Minds method accessible to anyone. And it's the one that could genuinely change how people make decisions — which means it's the one with the most potential to matter.

But I'd build all three. The question is sequence, not selection.

— Steve
