## Thread 3: Feature Deep Dive

### Tweet 1 (Hook)
```
Most AI coding tools help you write code.

IGNITE helps you decide WHAT to code.

Here's how the debate actually works:
```

### Tweet 2 (The Agents)
```
You're not talking to one AI. You're hosting a conversation between:

- Steve Jobs: Design, emotion, user experience
- Elon Musk: Physics, speed, competitive moats
- Phil Jackson: Synthesis, conflict resolution

Each has a distinct perspective.
```

### Tweet 3 (How It Works)
```
Drop your PRD:

/agency-debate my-idea.md

The system distributes your PRD to the debate agents.

They read it independently, then engage in a structured discussion.
```

### Tweet 4 (The Debate)
```
The debate has three phases:

1. Opening positions (2-3 min) — Each states their view
2. Direct engagement (5-7 min) — They challenge each other
3. Synthesis (3-5 min) — Phil consolidates into decisions
```

### Tweet 5 (What You Get)
```
The output is a synthesis document:

✓ Locked decisions (stop debating, start building)
✓ Build priorities (P0/P1/P2)
✓ Open questions (your homework)
✓ Risk register (what could go wrong)
```

### Tweet 6 (Why It Works)
```
This works because:

1. Disagreement surfaces blind spots
2. Synthesis forces decisions
3. Priorities emerge from tension, not consensus

The best feedback comes from people who disagree thoughtfully.
```

### Tweet 7 (CTA)
```
Try the demo:

/agency-demo

90 seconds. No API key.

See what your idea would look like through their eyes.
```

## Engagement Tips

1. **Include screenshots** — Debate output is visually interesting
2. **Quote the agents** — Their quotes are often the hook
3. **Be specific** — "My link saver app" not "my idea"
4. **Show results** — Deployed URLs beat abstract promises
5. **Invite replies** — "What would you debate?" drives engagement
